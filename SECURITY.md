# Security Policy

We take the security of **Access Control Kit** seriously. If you believe you have found a security vulnerability, please report it to us responsibly using the process outlined below.

---

## Supported Versions

Only the latest major and minor releases receive security updates. We actively encourage all developers to upgrade to the latest versions promptly.

| Version | Supported |
| :--- | :---: |
| v0.3.x | Yes |

---

## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a vulnerability, please email us directly at [akshad.dhole14@gmail.com](mailto:akshad.dhole14@gmail.com) (or use our private reporting form on GitHub if configured).

Please include the following details in your report:
- A description of the vulnerability and its potential impact.
- Detailed step-by-step reproduction instructions (or a minimal proof-of-concept script).
- Any details regarding your local environment.

---

## Response Timeline

- **Acknowledgement**: We aim to acknowledge your report within 48 hours of receipt.
- **Triage**: We will verify the report and determine the impact within 5 business days.
- **Fix**: If a vulnerability is confirmed, we will coordinate a fix and release a patched version as soon as possible.
- **Disclosure**: We will coordinate the public announcement of the vulnerability after a patch has been made available to consumers.
